### #67｜二进制求和

- 题型 / 标签：字符串、模拟、双指针、进位
- 正确思路：两个指针从两个字符串末尾向前遍历；越界补 `0`；每轮计算 `sum = aNum + bNum + carry`，用 `sum % 2` 记录当前位，用 `sum / 2` 更新进位；最后反转结果。
- 关键不变量 / 模板：`while (i >= 0 || j >= 0 || carry > 0)`，循环结束前必须处理最后的进位。
- 时间复杂度：O(max(m, n))
- 空间复杂度：O(max(m, n))
- 易错边界：
  - 指针条件必须是 `>= 0`，否则会遗漏第 0 位。
  - 两个字符串长度不一致时，已越界的一侧应补 `0`。
  - 结果需要 `reverse()`。
- 今日错误：
  - 写成 `sb.append(sumNum % 2 + '0')`。表达式的结果是整数 `48/49`，会调用 `StringBuilder.append(int)` 并追加 `"48"` 或 `"49"`；应写为 `sb.append(sumNum % 2)`。
- 下次复习：2026-07-21（+1 天）

### #258｜各位相加

- 题型 / 标签：数学、数字拆解、模拟
- 正确思路：外层循环负责“结果是否仍为多位数”；内层循环通过 `num % 10` 取末位、`num / 10` 去末位，计算一次各位和。
- 关键不变量 / 模板：只有 `0 ~ 9` 才是最终的一位数；只要当前和 `>= 10` 就继续拆位。
- 时间复杂度：O(log n)
- 空间复杂度：O(1)
- 易错边界：
  - `0` 应直接返回 `0`。
  - `10` 仍是两位数，必须继续计算为 `1`。
- 今日错误：
  - 外层条件写成 `while (sum > 10)`，会漏掉 `sum == 10` 的情况；应改为 `while (sum >= 10)`。
- 下次复习：2026-07-21（+1 天）
